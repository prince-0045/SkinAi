# AI Model Architecture and Details

This document provides a comprehensive overview of the SkinAi AI model used for skin disease prediction, detailing its architecture, inputs, outputs, and the specific classes it can identify. It is designed to be presented to understand the complete working of the model.

## 1. Overview

The SkinAi project utilizes a Deep Learning Convolutional Neural Network (CNN) for image classification. Specifically, it employs the **MobileNet** architecture, a lightweight and efficient model designed for mobile and resource-constrained environments, while still providing high accuracy for computer vision tasks.

- **Base Architecture:** MobileNet (Pre-trained)
- **Task:** Multi-class Image Classification
- **Number of Classes:** 7

## 2. Input to the Model

The model expects an image of a skin lesion as input.

- **Format:** The raw input is image bytes (e.g., from a JPEG or PNG file uploaded by the user).
- **Preprocessing:**
  1.  **Decoding:** The image bytes are decoded into a Pillow (PIL) Image object and converted to RGB color format.
  2.  **Resizing:** The image is resized to a fixed dimension of **224 x 224 pixels**. This is the standard input size for the MobileNet architecture.
  3.  **Array Conversion:** The image is converted into a NumPy array of shape `(224, 224, 3)`, where '3' represents the Red, Green, and Blue color channels.
  4.  **Batching:** A batch dimension is added, making the shape `(1, 224, 224, 3)` because the model expects a batch of images, even if it's just one.
  5.  **Scaling/Normalization:** The pixel values are preprocessed using `keras.applications.mobilenet.preprocess_input`. For MobileNet, this typically involves scaling the pixel values from the range `[0, 255]` to the range `[-1, 1]`.

## 3. Model Architecture Details

The model is built using a "Transfer Learning" approach.

### 3.1. Base Model (Feature Extractor)

-   **Network:** `keras.applications.MobileNet`
-   **Weights:** The model uses custom pre-trained weights loaded from the `model.h5` file, not just the generic ImageNet weights.
-   **Structure:** MobileNet relies on **Depthwise Separable Convolutions**. This significantly reduces the number of parameters and computation compared to standard convolutions, making it fast and efficient.
-   **Pooling:** The final output of the base MobileNet layers (the feature maps) goes through 'Global Average Pooling' (`pooling='avg'`). This converts the spatial dimensions (height x width) into a single 1D vector of features for the whole image.

### 3.2. Custom Classification Head

On top of the MobileNet feature extractor, custom layers are added to perform the specific 7-class classification for skin diseases:

1.  **Dropout Layer:** A `Dropout` layer with a rate of `0.2`.
    -   *Function:* This is a regularization technique. During training, it randomly sets 20% of the input units to 0 at each update. This helps prevent the model from overfitting onto the training data, making it generalize better to new, unseen images.
2.  **Dense output Layer (Fully Connected):**
    -   *Units:* `7` (corresponding to the 7 skin disease classes).
    -   *Activation Function:* **`softmax`**
    -   *Function:* The `softmax` activation function is crucial for multi-class classification. It takes the unnormalized outputs (logits) from the previous layer and converts them into a probability distribution. The output is a vector of 7 values, each between 0 and 1, and the sum of all 7 values equals exactly 1.0. Each value represents the model's confidence probability that the image belongs to that specific class.

## 4. Output of the Model

The direct output of the neural network is an array of 7 probabilities, one for each disease class.

### 4.1. Post-processing

1.  **Argmax:** The system finds the index of the highest probability in the output array (`np.argmax`).
2.  **Class Mapping:** This winning index is mapped to the corresponding disease name from the predefined list of classes.
3.  **Confidence Score:** The probability value at the winning index is extracted as the model's "confidence" score.

### 4.2. Final API Response returned by the backend

The final output generated by the application after running the model combines the prediction with medical information. It returns a structured JSON object containing:

-   `disease`: The predicted name of the disease (e.g., "Melanoma").
-   `confidence`: A floating-point number between 0.0 and 1.0 indicating the model's certainty.
-   `severity`: An assessment of the condition's danger (e.g., "Critical", "Moderate", "Low").
-   `description`: A detailed medical explanation of the predicted condition.
-   `recommendation`: High-level advice on what the user should do next.
-   `do_list`: An array of actionable steps the user should take.
-   `dont_list`: An array of actions the user must avoid.

## 5. The 7 Classes (HAM10000 Dataset)

The model is trained to recognize the following 7 classes of skin lesions, based on the standard HAM10000 dataset:

1.  **Actinic Keratosis** (Precancerous)
2.  **Basal Cell Carcinoma** (Skin Cancer - highly treatable)
3.  **Benign Keratosis** (Non-cancerous growth)
4.  **Dermatofibroma** (Harmless skin nodule)
5.  **Melanoma** (Aggressive Skin Cancer - Critical)
6.  **Melanocytic Nevi** (Common harmless mole)
7.  **Vascular Lesion** (Blood vessel abnormalities)

*(Note: The system also includes fallback logic for an "Unknown" state if the prediction confidence is too low or an error occurs).*
